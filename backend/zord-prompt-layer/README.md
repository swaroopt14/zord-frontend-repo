# Zord Prompt Layer

The Zord Prompt Layer is an AI-powered Text-to-SQL service that powers the "Ask Zord" Copilot. It translates natural language questions into safe, tenant-isolated SQL queries against the Zord databases (Intent Engine, Enclave, and Relay) and returns a formatted natural-language explanation of the results.

It uses the **Google Gemini API** for LLM orchestration and is completely isolated from PII, ensuring that no sensitive user data is ever exposed to third-party LLMs.

---

## ⚡ Quick Start (Local Development)

### 1. Prerequisites
Ensure you have the following installed on your machine:
* [Docker Desktop](https://www.docker.com/products/docker-desktop/) (and Docker Compose)
* [Go 1.22+](https://go.dev/) (if you want to run the automated test script natively)
* PowerShell (for running the `.ps1` seed script on Windows)

### 2. Configure Environment Variables
1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```
2. Open `.env` and configure your **Gemini API Keys**. The system supports round-robin API key rotation to handle Gemini's severe free-tier rate limits (429 errors).
   ```env
   # Provide multiple comma-separated keys to avoid 429 Rate Limit Errors
   GEMINI_API_KEYS=AIzaSy...key1,AIzaSy...key2,AIzaSy...key3
   ```
   > **Note:** If you only have one key, just set `GEMINI_API_KEY=AIzaSy...` directly.

### 3. Start the Core Dependencies
The Prompt Layer needs the exact same shared database network as the rest of Zord (`arealis-zord_zord-network`). 
First, ensure the core Zord platform is running:
```bash
cd ../zord-edge
docker compose up -d
```
*(This starts the PostgreSQL databases, Kafka streams, and Redis caches that the Prompt Layer will query).*

### 4. Start the Prompt Layer
Navigate back to the `zord-prompt-layer` directory and boot it up:
```bash
docker compose up -d --build zord-prompt-layer
```
The Prompt Layer API will now be available locally on port `8086`.

## 💻 Testing the Prompt Layer

Once the container is running and healthy on port `8086`, you can run the automated End-to-End (E2E) Test Script.

The test script will:
1. Connect directly to your local Postgres databases (`zord_intent_engine_db` & `zord_relay_db`).
2. Seed mock records (Payment Intents, Webhooks, Outbox items, Dead Letter Queue items, etc).
3. Send multiple natural language queries (both specific and general) to the Prompt Layer via HTTP.
4. Print the Markdown reasoning, confidence scores, and citations generated by Gemini.

**Run the test script via PowerShell:**
```powershell
.\test_prompt_layer.ps1
```

## 🛠️ How It Works (Text-to-SQL Architecture)
1. **Frontend Request**: The Zord Console (`zord-console`) sends the User's prompt, along with their actively selected `tenant_id` context, to `/query`.
2. **Dynamic SQL Generation**: The Prompt Layer provides Gemini with a strictly structured, non-PII database schema outline. Gemini writes a raw Postgres SQL query aimed at answering the question.
   - *Security Check*: **Every** generated SQL query is programmatically intercepted and forced to include `WHERE tenant_id = '{user_tenant}'`. It is impossible for one tenant to query another tenant's data.
3. **Database Execution**: The Go backend runs the SQL against the local PostgreSQL container network.
4. **Synthesis**: The raw SQL results are passed back to Gemini, which reads the JSON rows and synthesizes a professional, Markdown-formatted explanation.
5. **UI Rendering**: The JSON payload (Markdown + citations + confidence score) is sent back to the Zord Console for rendering in the Copilot UI.
